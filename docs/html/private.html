/confirm_user

Подтверждение учетной записи пользователя по коду из письма

/confirm_user get

get

Подтверждение учетной записи пользователя по коду из письма. Код сбрасывается при вызове данного метода в не зависисмости от его правильности, что защищает от перебора кодов. По логике метод должен быть POST, но так как вызов идёт из письма мы используем метод GET.

get /confirm_user

Подтверждение учетной записи пользователя по коду из письма. Код сбрасывается при вызове данного метода в не зависисмости от его правильности, что защищает от перебора кодов. По логике метод должен быть POST, но так как вызов идёт из письма мы используем метод GET.

Query Parameters

  • user_id: required (integer)

    Идентификатор пользователя

    1
  • confirm_code: required (string)

    Код подтверждения

    Example:

    aabbccddeeff00112233445566778899

/forgot

Восстановление пароля пользователя

/forgot post

post

Запрос письма с новым паролем для указанного аккаунта. Чтобы указанный в письме пароль стал актуальным необходимо перейти по ссылке из письма.

post /forgot

Запрос письма с новым паролем для указанного аккаунта. Чтобы указанный в письме пароль стал актуальным необходимо перейти по ссылке из письма.

  • Request

Query Parameters

  • email: required (string - maxLength: 129)

    Email адрес, на который зарегистрирован аккаунт

    Example:

    test@test.ru

/template

Шаблоны писем

/template get post put delete

get

Получаем список писем или одно письмо. В зависимости от наличия параметра id будет выведена либо одна запрошенная запись, либо список.

post

Добавение нового пустого шаблона (обязательно вызываем этот метод при открытие конструктора, чтобы был идентификатор шаблона, к которому будем привязывать картинки). Параметры запроса передавать не нужно.

put

Редактирование шаблона

delete

Удаление шаблона

get /template

Получаем список писем или одно письмо. В зависимости от наличия параметра id будет выведена либо одна запрошенная запись, либо список.

Secured by cookie
  • Request
  • Security

Query Parameters

  • id: (integer)

    Идентификатор шаблона, исключает возможность применения всех остальных параметров запроса

    Example:

    1
  • sort: (one of dt_add, dt_last_use - default: dt_last_use)

    Поле по которому будет отсортирована выборка

    Example:

    dt_last_use
  • sort_order: (one of asc, desc - default: desc)

    Направление сортировки

    Example:

    desc
  • limit: (integer - default: 10 - minimum: 0 - maximum: 100)

    Число записей на странице (случае отклонения переданного значения от заданных границ, limit будет автоматиески приведен к ближайшей)

    Example:

    30
  • offset: (integer - default: 0 - minimum: 0)

    Смещение выборки на указанное число записей (в случае отклонения переданного значения от заданных границ, limit будет автоматиески приведен к ближайшей)

    Example:

    0
